# Motivation

## 背景：产品配置求解器的挑战

在现代制造业和服务行业中，产品配置是一个核心场景。从标准 T 恤的尺寸选择，到复杂的工业设备参数搭配，用户需要根据各种约束条件来确定最终的配置方案。

传统做法中，这种配置通常由人工完成：
- 销售工程师根据经验判断
- 查阅大量的规则文档
- 反复沟通确认参数

但当约束条件变得复杂时，人工方式面临巨大挑战：
- 规则之间的依赖关系难以梳理
- 多参数组合的可行性判断耗时
- 配置错误导致的生产损失

## 现有方案的局限

### 1. 硬编码规则

将业务规则直接写入代码：

```java
if (color.equals("Red") && size.equals("Small")) {
    qty = 1;
}
```

这种方式的问题：
- 规则分散在代码各处，难以维护
- 添加新规则需要修改代码
- 无法可视化和文档化规则逻辑

### 2. 规则引擎

使用 Drools 等规则引擎：

```drools
rule "TShirt Rule"
when
    $c: Color(color == "Red")
    $s: Size(size == "Small")
then
    insert(new TShirt(qty = 1));
end
```

虽然比硬编码灵活，但：
- 规则执行效率有限
- 多实例场景支持不足
- 缺乏全局优化能力

### 3. 数据库查询

将规则存储在数据库中，通过查询匹配：

```sql
SELECT * FROM rules WHERE color = 'Red' AND size = 'Small'
```

问题：
- 复杂约束难以表达
- 无法处理数值关系
- 查询性能随规则数量下降

## 解决方案：基于约束求解的参数反推

**JMix Config Engine** 采用 **约束满足问题 (CSP)** 的方法，将配置问题转化为数学模型，利用高效的 SAT 求解器自动计算满足所有约束的参数组合。

> **关于 "Mix" 的含义**：JMix 中的 "Mix" 代表**混合策略**。我们认为，没有一种方法是万能的：
> - **约束求解**擅长处理复杂的组合优化问题，但面对循环、递归等场景时有局限
> - **传统代码执行**可以灵活处理任意逻辑，但难以保证全局最优
>
> JMix 的设计理念是：**让合适的算法做合适的事**。复杂的多层约束交给 SAT 求解器；简单的循环逻辑用代码直接执行。两者融合，兼顾求解能力与执行灵活性。

### 核心技术：SAT 与 CP-SAT

**SAT（全布尔可满足性问题）** 是计算机科学中的经典问题：判断一个布尔公式是否可满足。CP-SAT 则在此基础上增加了整数变量和线性约束的支持，非常适合配置求解领域。

CP-SAT 的优势：
- **成熟高效**：Google OR-Tools 的 CP-SAT 求解器经过多年优化，在工业界广泛应用
- **规模强大**：支持百万级变量规模，毫秒级求解
- **全局最优**：不是局部推理，而是全局搜索最优解
- **冲突诊断**：无解时自动定位冲突约束

### 核心思路

将业务规则转化为约束表达式，由求解器自动探索解空间：

```java
// 定义参数变量
ParaVar colorVar = new ParaVar("color", {"Red", "Black", "White"});
ParaVar sizeVar = new ParaVar("size", {"Small", "Medium", "Big"});
PartVar tShirt = new PartVar("TShirt", price = 100);

// 定义约束规则
addConstraint_rule2() {
    // 如果颜色是红色且尺寸是小号，则TShirt数量为1；否则为3
    if (colorVar == "Red" && sizeVar == "Small") {
        tShirt.qty == 1;
    } else {
        tShirt.qty == 3;
    }
}
```

### 技术特点

1. **毫秒级复杂计算**
   基于 CP-SAT 求解器，即使是复杂的多层约束，也能在毫秒内完成求解

2. **百万变量规模**
   支持大规模约束模型，适用于工业级配置场景

3. **完善的冲突诊断**
   当约束无解时，自动定位冲突规则，直观展示问题根源

4. **丰富的约束类型**
   - **兼容性约束**：Requires、CoDependent、Incompatible
   - **计算约束**：数值关系、条件表达式
   - **选择约束**：单选、多选组合
   - **优先级约束**：多目标优化

5. **多实例支持**
   支持动态多实例场景，如多件产品的配置求解

6. **差量加载**
   通过依赖图调度机制，只加载相关规则，提高大规模场景的性能

## 典型应用场景

1. **产品配置器**：帮助客户选择满足约束的产品参数
2. **订单参数反推**：已知部件数量，反推配置参数
3. **资源分配**：多目标优化的资源配置
4. **配置验证**：批量验证已有配置的有效性

## 解决的问题

JMix Config Engine 旨在解决以下问题：

- **规则复杂难维护**：声明式规则定义，规则即文档
- **人工配置易出错**：自动求解，消除人为判断失误
- **求解效率低下**：CP-SAT 求解器保证毫秒级响应
- **冲突定位困难**：自动诊断，直观展示冲突根源
- **规模受限**：支持百万级变量，满足工业级需求

## 目标用户

本项目面向 **后端开发集成者**，提供高性能的配置计算引擎，帮助你快速集成约束求解能力到业务系统中。

规则管理、配置界面等上层应用不在本项目范围内——JMix 专注于做好一件事：**计算**。
