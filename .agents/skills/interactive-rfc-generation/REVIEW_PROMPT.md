# RFC AI 评审检查清单

当需要AI评审RFC文档时，使用以下检查清单。

---

## 评审角度

### 1. 完整性检查

- [ ] 是否包含所有必需章节（摘要、动机、设计方案、验收准则、实现计划、参考资料）
- [ ] 摘要是否清晰描述问题和解决方案
- [ ] 动机部分是否包含具体的问题场景和示例
- [ ] 设计方案是否覆盖所有边界情况
- [ ] 验收准则是否包含具体可执行的测试用例

### 2. 一致性检查

- [ ] 动机中的问题场景与设计方案是否对应
- [ ] 验收准则中的测试用例是否覆盖设计方案中的关键路径
- [ ] 实现计划中的任务是否与设计方案一致
- [ ] 术语使用是否统一（检查文档内部）
- [ ] 与项目现有设计（CORE-DESIGN.md）是否一致

### 3. 可实现性评估

- [ ] 设计方案是否过于复杂，是否可以简化
- [ ] 是否有潜在的向后兼容性问题
- [ ] 实现计划中的优先级是否合理
- [ ] 是否有遗漏的依赖关系
- [ ] 测试用例是否可以实际执行

### 4. 清晰度评估

- [ ] 术语定义是否清晰
- [ ] 流程图或文字描述是否易于理解
- [ ] 代码示例是否完整可运行
- [ ] 错误处理是否有明确说明

---

## 评审输出格式

```markdown
## RFC 评审报告

**RFC编号**: RFC-XXXX
**评审日期**: YYYY-MM-DD
**评审者**: AI评审

### 整体评价

[总体评价：优秀/良好/需要改进]

### 1. 完整性评分

| 章节 | 评分 | 说明 |
|------|------|------|
| 摘要 | X/10 | ... |
| 动机 | X/10 | ... |
| 设计方案 | X/10 | ... |
| 验收准则 | X/10 | ... |
| 实现计划 | X/10 | ... |

**综合评分**: X/60

### 2. 发现的问题

#### 🔴 严重问题

1. [问题描述]
   - 位置：[章节]
   - 建议：[如何修复]

#### 🟡 建议改进

1. [改进建议]

### 3. 一致性问题

- [列表问题]

### 4. 可实现性评估

- [评估结果]

### 5. 改进建议

[按优先级排序的具体改进建议]

### 6. 结论

[是否建议批准/需要修改后重新评审]
```

---

## 评审执行步骤

1. 读取 RFC 文档全文
2. 对照检查清单逐项检查
3. 记录发现的问题和改进建议
4. 生成评审报告
5. 输出到对话中或保存到 `.specstory/review/RFC-XXXX-review.md`

---

## 评审注意事项

- 保持客观，基于文档内容进行评价
- 重点关注可执行性和测试覆盖
- 指出问题时同时提供改进建议
- 确保与项目整体设计保持一致
