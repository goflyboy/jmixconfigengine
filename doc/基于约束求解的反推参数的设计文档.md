# 基于约束求解的反推参数的设计文档

## 1. 概述

### 1.1 设计目标
设计一个基于约束求解的反推参数系统，能够根据已知的部件数量和参数值，通过约束求解技术反推出其他未知参数的值。该系统采用图调度机制实现差量加载，支持多种约束类型和复杂的业务规则。

### 1.2 核心功能
- 基于Ortools SAT约束求解器的参数反推
- 支持差量加载和全量加载两种模式
- 提供图调度机制优化求解性能
- 支持多种约束类型（兼容性、计算、选择等）
- 提供完整的南向扩展接口

### 1.3 使用流程
1. 初始化约束执行器并添加模块
2. 构建约束模型和变量映射
3. 通过图调度机制确定需要加载的规则和变量
4. 执行约束求解获取参数反推结果
5. 后处理优化和结果验证

## 2. 架构设计

### 2.1 整体架构
```
北向接口 (ModuleConstraintExecutor)
    ↓
执行器实现 (ModuleConstraintExecutorImpl)
    ↓
约束算法基类 (ConstraintAlgImpl)
    ↓
CP模型封装 (AlgCPModel)
    ↓
Ortools SAT求解器
```

### 2.2 核心组件
- **ModuleConstraintExecutor**: 北向接口，提供统一的约束求解服务
- **ModuleConstraintExecutorImpl**: 执行器实现，管理模块和求解流程
- **ConstraintAlgImpl**: 约束算法基类，提供南向扩展接口
- **AlgCPModel**: CP模型封装，简化Ortools API调用

## 3. 详细设计

### 3.1 ModuleConstraintExecutor设计

#### 3.1.1 核心职责
- 提供全局单例访问点
- 封装内部实现细节
- 提供统一的约束求解接口

#### 3.1.2 主要接口
- `init(ConstraintConfig)`: 初始化执行器
- `addModule(Long, Module...)`: 添加约束模块
- `inferParas(InferParasReq)`: 执行参数推理
- `registerExtensible(ExtensibleProcess)`: 注册扩展处理器

#### 3.1.3 使用示例
以ModuleConstraintExecutorTest.testInitAddModuleAndInfer为例：

```java
// 1. 初始化执行器
ModuleConstraintExecutorImpl exec = new ModuleConstraintExecutorImpl();
ConstraintConfig cfg = new ConstraintConfig();
cfg.isAttachedDebug = true;
exec.init(cfg);

// 2. 加载模块
Module m = ModuleUtils.fromJsonFile("tshirtdata.json");
m.init();
exec.addModule(123123L, m);

// 3. 执行参数推理
InferParasReq req = new InferParasReq();
req.moduleId = 123123L;
req.enumerateAllSolution = true;
Result<List<ModuleInst>> result = exec.inferParas(req);
```

## 4. 图调度机制

### 4.1 设计原理
图调度机制是系统的核心优化技术，通过构建规则依赖关系图，实现差量加载和智能调度。

#### 4.1.1 依赖关系图
- **节点**: 编程对象(progObjCode)
- **边**: 规则(edgeRuleCode)
- **方向**: 从左侧节点到右侧节点

#### 4.1.2 图结构
```
fromToEdges: 正向依赖映射
toFromEdges: 反向依赖映射
allNodes: 所有节点集合
allRefProgObjMap: 节点到编程对象的映射
```

### 4.2 查询子图算法

#### 4.2.1 算法流程
```
输入目标编程对象编码
    ↓
初始化结果集合
    ↓
递归查找依赖关系
    ↓
收集相关规则和变量
    ↓
返回子图信息
```

#### 4.2.2 依赖查找
- 使用深度优先搜索遍历依赖图
- 防止循环依赖和重复访问
- 收集所有相关的规则和编程对象
- 支持多目标查询

### 4.3 差量加载优化

#### 4.3.1 加载策略
- 根据输入参数确定需要加载的规则
- 只初始化相关的编程对象
- 减少不必要的约束和变量
- 提高求解性能

#### 4.3.2 性能优势
- 减少内存占用
- 提高求解速度
- 支持大规模约束模型
- 降低计算复杂度

## 5. 约束类型详解

### 5.1 兼容性约束

#### 5.1.1 Requires约束
- **定义**: 如果左侧参数选择指定选项，则右侧参数必须选择指定选项
- **实现**: 使用布尔逻辑和蕴含关系
- **示例**: (a1,a3) Requires (b1,b2,b3)

#### 5.1.2 CoDependent约束
- **定义**: 双向依赖关系，左右参数必须在对应组内或组外
- **实现**: 双向蕴含关系
- **示例**: (a1,a3) CoDependent (b1,b2,b3)

#### 5.1.3 Incompatible约束
- **定义**: 双向不兼容关系，左右参数不能在对应组内同时存在
- **实现**: 反向蕴含关系
- **示例**: (a1,a3) Incompatible (b1,b2,b3)

### 5.2 计算约束

#### 5.2.1 条件计算
- **定义**: 基于条件表达式的数值计算
- **实现**: 使用布尔变量和条件约束
- **示例**: if(condition) then value = expr1 else value = expr2

#### 5.2.2 数值关系
- **定义**: 参数之间的数值关系约束
- **实现**: 使用线性约束
- **示例**: part1.qty = para1.value + para2.value

### 5.3 选择约束

#### 5.3.1 单选约束
- **定义**: 确保参数只能选择一个选项
- **实现**: 使用ExactlyOne约束
- **示例**: 颜色参数只能选择红、黑、白中的一种

#### 5.3.2 多选约束
- **定义**: 允许参数选择多个选项
- **实现**: 使用AtLeastOne或AtMostOne约束
- **示例**: 功能参数可以选择多个功能组合

1. **图调度机制**: 通过依赖关系图实现差量加载，提高求解性能

该系统为产品配置器提供了强大的参数反推能力，能够处理复杂的业务约束和规则，支持大规模约束模型的求解，为产品配置和参数优化提供了重要的技术支撑。
